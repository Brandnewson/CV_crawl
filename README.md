# cv-harvester

An agentic Claude Code tool that mines your code projects, GitHub history, and job
applications to produce tailored CV bullets and cover letters for each role you apply to.
Runs entirely locally — no external CV service, no data leaving your machine.

---

## What you get

| Command | What it does |
|---|---|
| `/cv-setup` | **Start here.** Guided interactive setup to populate your profile, work experience facts, and cover letter stories |
| `/cv-harvest` | Analyses all your local projects and GitHub history, producing a store of per-project CV entries (~3-5 min) |
| `/cv-discover` | Searches for new jobs using JobSpy, deduplicates, scores with OpenAI, and shows ranked results |
| `/cv-apply` | Takes a job from your PostgreSQL pipeline, asks targeted gap-fill questions, writes a tailored DOCX + PDF CV (~5-20 min) |

| Requirement | Min version | Check |

| PostgreSQL | 14+ | `pg_isready` |

### Install Python dependencies

```powershell
# From the CV_crawl directory
uv sync --project "C:\Code\CV_crawl"
```

Or install individually:
```powershell
pip install pypdf mammoth python-docx psycopg2 lxml pydantic python-dotenv openai anthropic
```

## Quick start for new users

```powershell
# 1. Install Claude Code
npm install -g @anthropic-ai/claude-code

# 2. Log in
claude login

# 3. Navigate to the repo
cd "C:\Code\CV_crawl"

# 4. Run the tool
claude

# 5. Inside Claude Code, run:
#    /cv-setup   ← populate your profile and facts
#    /cv-harvest ← analyse your projects
#    /cv-apply   ← generate a tailored CV
```

The first time, start with `/cv-setup` — it walks you through creating all the fact
files the pipeline depends on. You only need to do this once (or when your circumstances
change).

## File structure

| File | Purpose | Edit manually? |
|---|---|---|
| `.cv-profile.json` | Your name, address, education, contact details | Yes — or use `/cv-setup` |
| `.cv-work-experience.json` | Verified facts per work experience role | Yes — or use `/cv-setup` |
| `.claude/skills/cover-letter-generation/story-bank.json` | Cover letter stories with facts | Yes — or use `/cv-setup` |
| `.cv-facts-promoted.json` | Reusable skill / tech facts (created by `/cv-setup`) | Yes |
| `.cv-harvest-store.json` | Project profiles (generated by `/cv-harvest`) | No — regenerate |
| `.experience-cache.json` | Per-job Q&A answers (auto-managed) | No — use `/cv-setup` option 5 to prune |
| `.jd-keywords-cache/` | Cached keyword extraction per job_id | No — safe to delete |
| `tools/cv_apply_contract.py` | Pipeline constants (paths, stage order) | Only when changing paths |

### Temporary files (auto-cleaned)

These are created during `/cv-apply` and deleted at the end of each run. If a run was
interrupted, you may find them in the working directory:

```
.cv-apply-evidence-pack-tmp.json   (large — safe to delete)
.cv-apply-slot-plan-tmp.json       (large — safe to delete)
.cv-apply-jd-keywords-tmp.json     (small — safe to delete)
.cv-apply-selections-tmp.json      (keep until cover letter is done)
.cv-apply-meta-tmp.json            (keep until cover letter is done)
```

## Customising for your own machine

The paths in this repo are hardcoded to the original developer's machine. If you are
setting this up fresh, update these files:

| File | What to change |
|---|---|
| `.claude/commands/cv-apply.md` | "Paths" table at the top — all `C:\Code\` and OneDrive paths |
| `.claude/commands/cv-harvest.md` | Job hunting folder path, GitHub username |
| `.claude/skills/cover-letter-generation/SKILL.md` | Output paths, profile path |
| `tools/cv_apply_contract.py` | `DEFAULT_CHECKPOINT_PATH`, `CANONICAL_FACT_STORES` paths |
| `tools/query_jobs.py` | Database connection string (if not using `.env`) |
| `profile/cv_template.docx` | Replace with your own CV template DOCX |
| `profile/scoring_profile.yaml` | Update target roles and keywords for your job search |

Also update your `.env` file (not committed) with your PostgreSQL and OpenAI credentials:
```env
DATABASE_URL=postgresql://user:password@localhost:5432/yourdb
OPENAI_API_KEY=sk-...
```

## How the pipeline works

```
User types: /cv-apply
                |
                v
     +----------------------+
     |   ORCHESTRATOR       |
     |   20 checkpointed    |
     |   stages             |
     +--+-------------------+
        |
   Evidence selection → Slot planning → CV Writer sub-agent
        |                                      |
        v                                      v
  .cv-apply-evidence-pack-tmp.json    .cv-apply-selections-tmp.json
        |                                      |
        v                                      v
  Deterministic validator          Render DOCX → PDF
        |                                      |
        v                                      v
  Layout gate (2-page check)       DB update + cover letter
```

### The 20 pipeline stages

| Stage | What happens |
|---|---|
| `job_select` | Query DB, display job list, user picks a job |
| `jd_extract` | Clean and extract keywords from the job description |
| `source_load` | Load work experience, projects, experience cache |
| `project_select` | Choose which projects to show on the CV |
| `gap_detect` | Find JD keywords not covered by existing facts |
| `gap_normalize` | Deduplicate and normalise gap questions |
| `evidence_select` | Map facts to CV slots via `evidence_select.py` |
| `slot_plan` | Build bullet intent cards via `slot_plan.py` |
| `draft_work_experience` | CV Writer writes work experience bullets |
| `draft_technical_projects` | CV Writer writes project bullets |
| `assemble` | Combine all sections |
| `validate_deterministic` | Check lengths, verbs, constraints |
| `render_docx_pdf` | Render DOCX using the template |
| `layout_gate_2pages` | Check the CV fits on 2 pages |
| `preview_feedback` | Show CV to user for approval |
| `feedback_classify` | Classify user feedback (style vs fact vs structure) |
| `fact_patch_apply` | Apply any fact corrections |
| `targeted_regen` | Regenerate only changed bullets |
| `persist_db` | Save output paths to DB |
| `cover_letter_handoff` | Optionally invoke the cover letter skill |

Each stage writes a checkpoint — if the process is interrupted, resume with:
```
uv run --project "C:/Code/CV_crawl" \
    python "C:/Code/CV_crawl/tools/cv_apply_runner.py" --resume
```

## Troubleshooting

| Problem | Fix |
|---|---|
| `claude: command not found` | Check PATH after `npm install -g @anthropic-ai/claude-code` |
| `uv: command not found` | Install uv: `pip install uv` or follow https://docs.astral.sh/uv |
| `pypdf` or `mammoth` import errors | `pip install pypdf mammoth` |
| GitHub rate limit in `/cv-harvest` | Set `GITHUB_TOKEN` env var for 5000 req/hr |
| OneDrive path errors | Verify with `ls "C:\Users\<you>\OneDrive..."` |
| DB not running | Start PostgreSQL: `pg_ctl start` or via your system service |
| Evidence pack empty | Run `/cv-harvest` first — `.cv-harvest-store.json` must exist |
| `.cv-work-experience.json` not found | Run `/cv-setup` first |
| Cover letter more than 1 page | Automatic: `enforce_one_page_cover_letter.py` handles this |
| Pipeline stuck at a stage | Delete `.cv-apply-checkpoint.json` to force a full restart |
| Wrong keywords extracted | Delete `.jd-keywords-cache/{job_id}.json` to force re-extraction |

## Limitations

- **Windows only** — paths are hardcoded to `C:\` and OneDrive. Linux/Mac would require
  path updates throughout (see "Customising" section above).
- **Requires PostgreSQL** — the job pipeline reads and writes to a local DB. A future
  version could support a flat-file fallback.
- **GitHub API** — `/cv-harvest` makes up to 90 API calls. Set `GITHUB_TOKEN` to avoid
  rate limits.
- **Microsoft Word** — the PDF render step uses Word COM automation. Word must be
  installed on the machine.
- **Token usage** — a full `/cv-apply` run consumes ~80-150K tokens. High-tier Claude
  Code usage is recommended for regular use.
